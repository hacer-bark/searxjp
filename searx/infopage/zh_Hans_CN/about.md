# 关于 SearXJP

SearXJP 是一个注重隐私强化、并针对亚洲地区优化的 [SearXNG] 元搜索引擎实例。

与标准的公共实例不同，SearXJP 是基于 **“知识分割架构 (Split-Knowledge Architecture)”** 构建的。我们优先考虑两件事：为日本和亚洲用户提供低延迟访问，以及在物理上 **隔离** “知道你是谁的服务器” 和 “知道你在搜什么的服务器”。

## 我们的架构：“知识分割” 模型

要在 SearXJP 上通过技术手段侵害您的隐私，攻击者必须同时攻破位于不同司法管辖区、由不同提供商拥有的两台服务器。

我们执行严格的 **“双节点策略”**：

### 1. 入口节点（日本东京）
这是您连接的服务器。它严格作为一个 **TCP 盲中继 (TCP Blind Relay)** 运行。
*   **它知道什么：** 您的 IP 地址。
*   **它不知道什么：** 它 **无法** 看见您的搜索查询。
*   **工作原理：** 该服务器将加密的 SSL 数据包直接转发到我们的后端。它不持有 SSL 密钥，这意味着该服务器解密或读取您的流量在 **数学上是不可能的**。

### 2. 处理节点（位置未公开）
该服务器通过加密的专用隧道接收来自日本入口节点的流量。
*   **它知道什么：** 它解密流量并知道您正在搜索什么。
*   **它不知道什么：** 它 **无法** 看见您的 IP 地址。它只能看到日本入口节点的内部 IP。
*   **工作原理：** 该节点负责计算处理，汇总来自 Google/Bing/DuckDuckGo 的结果，并在通过隧道发回之前对数据进行清洗（Sanitize）。

## 网络性能与中国大陆的可访问性

我们的东京入口节点采用优质的 BGP 网络组合，包括 **GSL, PCCW, COGENT, EIE, 和 JPNAP**。

**对于中国大陆用户：**
虽然我们尚未对穿越防火墙（Great Firewall）的可访问性进行正式测试，也无法保证完全可用，但我们的网络基础设施在技术上针对跨境连接进行了优化。由于接入了 **PCCW** 及其他优质亚洲线路，理论上相比连接欧美节点，我们能提供更佳的延迟和稳定性。

## 透明度：日志与数据保留

我们相信彻底的透明度，特别是关于我们记录什么，以及更重要的——我们 **无法** 记录什么。

### 标准互联网连接 (Clearweb)
*   **在日本入口节点上：** 我们保留滚动的 **15分钟 TCP 连接日志**。
    *   *原因：* 这严格用于 DDoS 防御和速率限制。如果某个 IP 向我们发起洪水攻击，防火墙会将其拦截。
    *   *隐私：* 由于该节点仅处理加密的 TCP流，**这些日志不可能包含您的搜索查询。**
*   **在处理节点上：** **无日志 (No Logs)。**
    *   我们不记录 IP 地址（我们看不到它们）。
    *   我们不记录搜索查询。
    *   速率限制是基于“路由”（例如：“全球流量”）应用的，而不是基于用户 IP。

### Tor 和 I2P 连接
*   **严格的无日志政策：** 通过 Tor 或 I2P 进入的流量完全绕过标准的 TCP 速率限制日志。
*   我们不追踪匿名网络的进入时间、相关数据或连接元数据。

## 如何设置为默认搜索引擎

SearXJP 支持 [OpenSearch]。您可以轻松将其添加到浏览器：

- [Firefox] - 右键点击地址栏，选择“添加 SearXJP”。
- [Chromium]内核浏览器 (Chrome, Brave, Edge) -以此进入 设置 > 搜索引擎 > 管理搜索引擎，如果未自动出现，请手动添加 SearXJP。
